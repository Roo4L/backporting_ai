# write-digest

Write a digest about the articles that you are being asked about.

The digest is supposed to provide the do the following things:

1. Highlight all the articles, OSS releases, blog posts, etc. that are being summarized by the digest. If there are duplicates, let's deduplicate them and only mention once. The goal is to give users persepctive of all updates that happened. Mention all arfifacts under one of three categories: Research, OSS, Blogs.
2. Provide summary about the most significant artifacts in each category: Research, OSS, Blogs. Something like this one:

    ```
    ## Research

    ### Title about Paper 1

    Paper 1 summary

    ### Title about Paper 2

    Paper 2 summary

    ## OSS

    ### OSS release 1 title

    OSS release 1 summary

    ...
    ```
